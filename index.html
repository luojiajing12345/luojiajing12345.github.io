全局对象window

当我们没有写任何代码的时候，里面就有个window对象，对象里有一堆函数(对象)。
浏览器默认全局对象window，无论在哪个网页都可以访问一个对象那就是window。
在浏览器中，window对象有双重角色，即是通过Javascript访问浏览器窗口的一个借口，有时ECMAScript规定的Globla对象。

由于window对象同时扮演着ECMAScript中的Global对象的角色，因此所有在全局作用域中声明的变量、函数都会变成window对象的属性和方法。
window对象下面一堆函数，对应一堆对象，对象下面又有对象，所有window下面很多对象。window里的函数前面可以不加window，例如window.alert(),直接alert()。
window下面的属性分为两类。

  1. ECMAScript规定的函数。
    parseInt、parseFloat、Number()（默认0）、String()（默认’’）、boolean()（默认false）、Objict(){默认{}}，window.setTineout(function(){} , time)(在指定时间后执行这个函数)
  2. 私有的函数（Chrome/ie/Fireox，像这些浏览器独有的，或者只有哪两个有的）。
    alert(弹框提示)、prompt(用户填写)、comfirm(确认)、console(开发者)。像这些浏览器私有的，每个浏览器的实现效果都可能不一样，因为没有标准，例如alert(),每个浏览器弹出的位置，大小，形状等都不一样。
    document(文档)有自己的规范（W3C规定），规定了能对文档执行的一切操作。DOM.
    history(浏览器),获取浏览器的历史，访问记录什么的。BOM.

简单类型和对象的区别

Number()

Number(‘1’)
var n = new Number(1);1相当于{valueof():1}
var n1 = 1;var n2=new Number(1)这两个的区别在于内存不同，n1就是在stack里有个数值1，n2是在stack里有个ADDR，然后指向heap里的对象{[[PrimitiveValue]]：1}，原始值为1，里面是个哈希。我们调用n2.valueOf(),能获取原始的这个1，用n2.toString()能获取它的字符串。没有人用第二种方法。
但第一种实际上是不能用toString()的，但js之父设计了一个临时转换的技巧，当n1.toString()的时候，会出现一个临时对象，调用temp=new Number(n1);temp.toString();然后这个temp就像不存在一样，就获取到它转换成字符串的了
所以就算我们使用第一种那种简单的赋值，也能使用第二种里的所有属性。
var n=1;n.xxx=2;n.xxx等于几？undefined。首先在stack里有个1，然后第二句的时候，stack里有个temp的ADDR，然后会在heap里建个对应的临时对象，n的值是1，它的属性xxx对应2，{xxx:2},运行这句以后temp会被回收，临时对象也会一起被回收，第三句的时候，没有这个xxx，会新建一个，新建的对象的没有xxx属性所以等于undefined。简单点说就是运行第二句会有个临时新对象，并把2赋值给它的xxx属性，运行完后销毁，第三局又新建个对象。

对象：var a2 = new Boolean(true)
注意：if(a2){console.log('aaa')}，aaa是会被打印出来的，因为a2是对象，在Boolean里，除了那五个(‘’,NaN,0,null,undefined)以外，其他都是true。所以里面判断为true。

Object()

简单类型：var a={}
对象：var a2=new Object()
这两个是没有区别的，但是，不是相等的。a===a2为false。因为它们在stack里的地址是不相等的，在heap里的位置也不相等。由此推论：所有新声明的对象都不相等，除非把stack里的地址赋值给拎一个变量。

共有属性（原型）

  • 上面几个对象的共有属性，像那些对象，有一些属性是每个都有的，例如：toString(),valueOf()等，主要作用是不要浪费内存。像这些共有属性都在隐藏的proto属性里面。当对象调用toString()方法时，会先在自己的属性里找有没有这个属性，如果没有找到，就会去proto里面找，找到后就会调用。
  • 以上所说的是对于常规对象，对于其他对象，例如Number对象，String对象，Boolean对象。它们会有一些自己才有的公有属性，例如Number对象的特有的toFixed(),toExponential()，String对象的length()等，它们首先是在自己的属性里找，然后在第一个proto里（类型对象的共有属性），然后再到所有对象的共有属性（第二个proto）里找，如果都找不到就会输出undefined。
  • 如果这些共有属性没有被人引用的话，是会被垃圾回收掉的，所以有东西在一直引用它。Object.prototype就是共有属性（原型），Number.prototype就是Number对象的共有属性。obj.__proto__===Object,prototype,Num.__proto__===Number.prototype,Num.__proto__.__proto__====Object.prototype。
  • Number对象，String对象，Boolean对象都有两个proto,object只有一个（为所有对象的共有属性）。这种几个结点连接一个结点也叫原型链。
  • 当我们没有写任何代码的时候，里面就有个window对象（浏览器在内存里初始化好了），对象里有一堆函数(对象)。例如：Number函数（对象），String函数（对象），Boolean函数（对象），这些对象里有个prototype属性，这个属性引用这它们自己类型的共用属性（例Number函数的prototype引用着Number对象的共用属性）,然后这些属性里有个proto引用着总对象的共用属性。
  • 当我们写了一句代码var a=new String('hi')，a对应的stack里存了一个heap里的地址，然后在heap里面有块空间，写着
    {0:’h’,1:’i’,length:2,proto:String},这个proto引用String的共用函数。window里的String里的prototype和s里的proto都指向String的共用属性，它们没有什么很大的区别。唯一的区别就是，前者是浏览器写好的，一开始就有的，String.prototype是String的共用属性的引用（如果没有这个，String会被垃圾回收掉）。后者是自己写了代码后才有的，a.proto是String的共用属性的引用（因为我们要用到它的共用属性）

javascript基本类型和引用类型在内存中的值

内存基本概念

内存的生存周期：

  1. 分配所需的内存
  2. 内存的读与写
  3. 不需要时将其释放
    所有语言的内存生命周期都基本一致，不同的是最后一步在低级语言中很清晰，但是在像JavaScript等高级语言中，这一步是隐藏的、透明的。

js内存生命周期：

  1. 定义变量时就完成了内存分配
  2. 使用值的过程实际上是对分配内存进行读取与写入的操作，读取与写入可能是写入一个变量或者一个对象的属性值，甚至传递函数的参数。
  3. 内存的释放依赖GC机制（高级语言解释器嵌入的“垃圾回收器”）。

数据和代码在内存中的存放

js程序在运行的时候，需要内存空间存放代码和数据，所以分为代码区和数据区。而数据区又分为栈(stack)和堆(heap)两种内存空间。

  • 栈(stack)是有结构的，每个块区按照一定次序存放（先进后出），sack中主要存放一些基本的变量和对象的引用，存在栈中的数据大小与生存期必须是确定的。可以明确知道每个区块的大小，因此，stack的寻址速度要快于heap。它的存储方式是一行一行的和栈类似。简单数据类型都存在这（6种）。
  • 堆(heap)是没有结构的，数据可以任意存放。heap用于复杂数据类型（引用类型）分配空间，例如数组对象、object对象。它的存储方式跟堆类似。
    例var a = 1,首先变量a提升，此时a等于undefined，然后a放在代码区，1放在数据区的栈里。在js内部，所有数字都是以64位浮点数形式储存，所以这个1在内存里占了64位。ECMAScript规定。
    复制基本类型：var b = 2,也和上面一样，当b=a，那么就把a对应的在栈里的64位浮点数形式的1覆盖在b对应的栈的位置。
    js在存字符的时候，每个字符占16位。ECMAScript规定
    存对象：var a = {name:'luo',age:18},那么会把a放在代码区，然后a对应的栈内存里有个ADDR（地址），那个地址是堆内存中的一块空间的地址，这块空间大小可以变化，就可以不受影响的增加或减少对象里的东西了。
    复制对象：相当于把变量对应的栈里面的ADDR复制给另一个变量。

引用：当复制保存着对象的某个变量时，操作的是对象的引用（就是那个地址）。但在为对象添加属性时，操作的是实际的对象。
关于循环引用var a,a={self:a},a.self等于undefined。也就是得到的对象是{self:undefined},因为第二步把对象赋值给a的时候，对象里的a等于undefined，运算是从右到左的。
循环引用应该是var a = {},a.self=a,会得到一个{self:{…}},点开发现里面也是一个{self:{…}}，一直循环。因为我们把a的值赋值给self的value，然后我们打开对象a，发现里面的栈有个地址，说明是个对象，按着这个地址找到堆里的空间，发现里面的self对应的又是个地址，再按着这个地址，一直循环，一直都在那个空间里找。

引用类型面试题：
输入：var a={n:1};var b=a;a.x=a={n:2}。
输出alert(a.x);-->undefined,alert(b.x);-->[object Object]。。
首先前两步是没有问题的，生成一个对象，赋值给a，然后把a的地址赋值给b，此时a和b在stack中的值（地址）是一样的，假设为ADDR100,对应的heap中的对象也是一样的。
然后a.x=a={n:2},假设{n:2}的地址为ADDR200，浏览器读数据是从左到右的，运算是从右到左的。所以此时a的值为ADDR100，右边运算赋值后给中间a后，中间的a后此时对应的stack中的值为ADDR200,左边的a还是ADDR100，然后把中间那个a，也就是ADDR200,赋值给ADDR100中的对象中的key为x的value。
那么此时a是ADDR200那里的的对象，里面的对象为{n:2},b为ADDR100那里的对象，里面的对象为{n:1,x:{n:2}},第三步的赋值只是赋给了ADDR100对应的对象里面的key为x的value。
当使用alert(a.x)，会调用toString方法。所以alert(a.x);-->undefined,alert(b.x);-->[object Object]。

垃圾回收机制（GC）

Javascript具有自动垃圾回收机制。
如果一个对象没有被引用，他就是垃圾，将被回收。
var a={name:'a'};var b={name:'b'};a=b，a就变成垃圾，将被回收，至于什么时候回收是不确定的。
var fn = function(){};document.body.onclick=fn;fn=null;请问这个函数是不是垃圾？
答：不是，当函数赋值给fn的时候，fn的stack指向那个对象，fn赋值给document.body.onclick时，document指向body,body指向onclick，onclick指向那个对象。当fn等于null，document那条连接并没有改变，对象还是被引用。
如果onclick等于null的话，那么它就是垃圾了。
如果把页面关闭的话，它也变成垃圾了。因为页面关了，相当于document不见了，然后它们内部只有body–>onclick–>function(){}了。
在IE6里面这三个不算垃圾，它会把这三个对象一直留着，只要不关闭整个浏览器就会一直留着，这样垃圾会越来越多。我们要把这个垃圾丢掉的话，只能运行window.onunload = function(){document.body.onclick = null}这种代码，把所有的事件监听赋值为空，所有的，这就是内存泄漏。
内存泄漏就是由一些浏览器引起的bug，使一些该被标记为垃圾的东西没有被标记为垃圾，然后内存就会被永久的占用着，浏览器会占用越来越多内存，会变慢，甚至崩溃。

深拷贝和浅拷贝概念

深拷贝：var a=1;var b=a;b变，不影响a。所以一般说深拷贝和浅拷贝的时候不考虑基本类型，因为赋值都是深拷贝。我们只说对象。
浅拷贝：var a={name:'a'};var b=a;b.name='b'；此时a={name:’b’},把a复制给b后，b改变，a也改变，这就是浅拷贝。
深拷贝：a和b在stack里有不同的地址，在heap里的空间也不一样，假设在heap里的对象中还有地址，也必须是不同的，这是两条分开，独立，互不影响的两条线。b变，a不变。这就是深拷贝。

响应式Web设计-viewport

什么是viewport?

viewport是用户网页的可视区域，翻译为中文叫做“视区”。
<meta name="viewport" content="width=device-width, initial-scale=1.0,user-scalable=no,maximum-scale=1.0,minimum-scale=1.0">
宽度=设备宽度，不允许缩放，所有比例都是1比1.

六个属性：

  1. width:控制viewport的大小，可以指定的一个值，如600，或者特殊的值，如device-width为设备的宽度（相当于100%的css像素）
  2. height：和width相对应，指定高度。
  3. initial-scale：初始缩放比例，也即是当页面第一次load的时候缩放比例。1.0，就是1：1
  4. maximum-scale：允许用户缩放到的最大比例。1.0
  5. minimum-scale：允许用户缩放到的最小比例。1.0
  6. user-scalable：用户是否可以手动缩放。yes或no

touch触摸事件

触摸事件的种类，有以下几种。可以通过TouchEvent。type属性，查看到底发生的是哪一种事件。

  1. touchstart:用户开始触摸时触发，它的target属性返回发生触摸的元素节点。
  2. touchend:用户不在接触触摸屏时（或移除屏幕边缘时）触发，它的target属性与touchstart事件一致的，就是开始触摸时所在的元素节点。它的changedTouches属性返回一个TouchList实例，包含所有不在触摸的触摸点（即Touch实例对象）
  3. touchmove:用户移动触摸点时触发，它的target属性与touchstart事件一致。如果触摸的半径、角度、力度发生变化，也会触发该事件。
  4. touchcancel:触摸点取消时触发，比如在触摸区域跳出一个情态窗口（modal window）、触摸点离开了文档区域（进入浏览器菜单栏）、用户的触摸点太多，超过了支持的上限（自动取消早先的触摸点）。

查看设备是否支持触摸(特性检测)

  1. canvas.ontouchstart，如果显示为null，则为可触摸设备，表示支持该功能，不支持显示为undefined。canvas为变量名。
  2. 查看ontouchstart是否body的一个属性。例：'ontouchstart' in document.body显示true，代表是，那么可触摸。
  3. document.body.touchstart == undefinede为非触屏设备。document.body.touchstart == null为触屏设备。
    in，用于查看什么属性或者什么值在不在xxx里面，例如：查看一个key在不在哈希里。'key' in hash，在就输出true，不在就输出false。

Canvas绘图

Posted on 2018-05-17 |

1.什么是Canvas?

<canvas>是html5新增的元素。这个元素负责在页面中设定一个区域，然后就可以通过JavaScripe动态地在这个区域中绘制图形。标签只是图形容器。

2.基本用法

  1. 创建一个画布。
    <canvas id="myCanvas" width="200" height="100" style='border:1px solid red'></canvas>不加边框看不见画布的范围。
  1. 绘制图像

var c=document.getElementById("myCanvas");//把元素赋值给c
var ctx=c.getContext("2d");//创建context对象，getContext（’2d’）对象是内建的html5对象，可以绘制简单的2D图形，拥有多种绘制路径、矩形、圆形、字符以及添加图像的方法
ctx.fillStyle="#FF0000";//填充样式，可以设定颜色、渐变或方案
ctx.fillRect(0,0,150,75);//填充方式，四个值x,y,width,heigth。
ctx.strokeStyle = 'red';//描边样式，边框颜色等
ctx.strokeRect =(30,30,50,50);//描边方式，和填充一样，但两个是分开的，这里为两个矩形。

描边属性：

  • 线条宽度：lineWidth
  • 线末形状：lineCap
  • 线条相交方式：lineJoin
    要先设置好宽度等属性再画线，即lineWidth要在stroke后面
  1. 坐标
    坐标默认是从画布左上角开始算，左上角为（0，0），横轴为x，竖轴为y。
  1. 路径
    在Canvas上绘制路径，首先必须调用beginPath()方法，表示要开始绘制新路径。
    然后，我们将使用以下几种方法。
  • arc(x,y,radius,startAngle,endAngle,counterclockwise)：以（x,y）为圆心绘制一条弧线，弧线半径为radius,起始和结束角度分别为startAngle和endAngle，最后一个参数表示起始和结束角度是否按逆时针方向计算，false为顺时针。
  • arcTo(x1,y1,x2,y2,radius):从上一点开始绘制一条曲线，到(x2,y2)位置，并且以给定的半径radius穿过(x1,y2)
  • bezierCurveTo(c1x,c1y,c2x,c2y,x,y):从上一点开始绘制一条曲线，到(x,y)为止，并且以(c1x,c1y)和(c2x,c2y)为控制点
  • moveTo(x,y)将绘图游标移动到(x,y)，不画线。
  • lineTo(x,y)从上一点绘制一条线，到(x,y)为止。
  • rect(x,y,width,height):从点(x,y)开始绘制一个矩形，宽度和高度分别为width,height。这个方法绘制的是矩形路径。
    创建完路径后：
  • 绘制一条连接到路径起点的线条，可用closePath(),
  • 填充，fill()
  • 描边，stroke()
  • 创建剪切区域，clip()

flex布局

布局的传统解决方案，基于盒状模型，依赖display属性 + position属性 + float属性。它对于那些特殊布局非常不方便，比如，垂直居中就不容易实现。
2009年，W3C提出了一种新的方案—-Flex布局，可以渐变、完整、响应式地实现各种页面布局，目前已经得到所有浏览器的支持。Flex布局将成为未来布局的首选方案。

Flex布局是什么？

Flex 是 Flexible Box的缩写，意味“弹性布局”，用来为盒状模型提供最大的灵活性。
任何一个容器都可以指定Flex布局
.box{display: flex;}

行内元素也可以使用Flex布局。
.box{display: inline-flex;}

Webkit内核的浏览器，必须加上-webkit前缀。
.box{display:-webkit-flex;display:flex;}
注意：设为Flex布局以后，子元素的float、clear和vartical-align属性将失效

用flex设置元素绝对居中(给它的父元素加下面三行代码)

.box{
display:flex;/如果是内联元素使用inline-flex
align-items: center;/水平居中
justify-content: center;/垂直居中
}

background-color背景颜色。

background-size背景图片大小。例：background-size:100px 100px
background-repeat指定如何重复背景图像。例：background-repeat：repeat
background-origin指定背景图像的定位区域。例：background-origin:padding-box
background-clip背景图像的绘画区域。例：background-clip：border-box
background-attachment背景图像是否固定或者随页面的其余部分滚动。background-attachment：scroll
background-image要使用一个还是多个图像。例：background-image:url(./img/img.jpg)
这几个值在background:里不分先后
