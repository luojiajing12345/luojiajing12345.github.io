结构化编程

结构化程式设计（英语：Structured programming），一种编程典范。它采用子程序、程式码区块（英语：block structures）、for循环以及while循环等结构，来取代传统的 goto。希望借此来改善计算机程序的明晰性、品质以及开发时间，并且避免写出面条式代码。
满足下面3个语句，就是结构化编程。

  1. 一行一行执行
  2. 有条件控制语句if…else…
  3. 有循环控制语句while(exp)do…

什么是算法

在高德纳的著作《计算机程序设计艺术》里对算法的特征归纳：

  • 输入：一个算法必须有零个或以上输入量。
  • 输出：一个算法应有一个或以上输出量，输出量是算法计算的结果。
  • 明确性：算法的描述必须无歧义，以保证算法的实际执行结果是精确地匹配要求或期望，通常要求实际运行结果的确定的。
  • 有限性：依据图灵的定义，一个算法是能够被任何图灵完备系统模拟的一串运算，而图灵机只有有限个状态，有限个输入符号和有限个穿衣函数（指令）。而一些定义更规定算法必须在有限个步骤内完成任务。
  • 有效性：又称可行性。能够实现，算法中描述的操作都可以通过已经实现的基本运算执行有限次来实现。

什么是数据结构

就是数据的结构。
一般来说这样的：

  1. 我们要解决一个跟数据相关的问题。
  2. 分析这个问题，相处对应的数据结构
  3. 分析数据结构，想出算法

    数据结构和算法是相互依存，不可分开的。
    学习完排序算法，就能了解常见的数据结构。

算法大分类

  • 分治法：把一个问题分区成互相独立的多个部分分别求解的思路。这种求解思路带来的好处之一是便于进行并行计算。（例如把国家分为省，然后把省分为市这样一路分，每个地方不一样的管理方式）
  • 动态规则法：当问题的整体最优解就是有局部最优解组成的时候，经常采用的一种方式。（就目前的情况来看，如果当前是最好的，那么就用这个。）
  • 贪婪算法： 常见的近似求解思路。当问题的整体最优解不是（或无法证明是）由局部最优解组成，且对解的最优性没有要求的时候，可以采用的一种方法。（短视算法，选择当前较好的方法）
  • 线性规划法：
  • 简并法：把一个问题通过逻辑或数学推理，简化成与之等价或者近似的、相对简单的模型，进而求解的方法。

排序算法

冒泡排序（体育委员两两对比法）：例如要把3、5、9、1、7、6这六个数字从左往右为以小到大。

  1. 对比第一个和第二个，把小的放左边，大的放右边。
  2. 然后对比第二个和第三个，也是小的放左边，大的放右边。
  3. 就这样依次对比，比完第五个和第六个。此时的排序为3、5、1、7、6、9。此时最大的值已固定。
  4. 依照前面的方法，但最后一位数不用再比，此时的排序为3、1、5、6、7、9。此时后两位二位已固定。
  5. 再按照前面的方法，最后两位不用比，此时的排序为1、3、5、6、7、9。此时后三位已固定。
  6. 再按照前面的方法，后3位不用比，此时排序为1、3、5、6、7、9.此时后四位已固定。
  7. 再按照前面的方法，后四位不用比，此时排序为1、3、5、6、7、9.此时后五位已固定，
  8. 那么这几个数字从左往右从小到大为1、3、5、6、7、9.

选择排序（体育老师一指禅法）：例如要把6、3、2、4、5、1这6个数字从左往右以小到大。

  1. 找出个最小的，把这6个数字相互比较，找出最小的数字也就是1，把跟最前面的数字换位置，得到的排序为1、3、2、4、5、6，此时最小值已固定。比较方法，例：6和3比，3小；3和2比，2小；2和4比。4小，这样。
  2. 从后五位中选出最小的，也就是2，和排在第二位的3换位置，得到的为1、2、3、4、5、6，此时前二位已固定。
  3. 从后四位中选出最小的，也就是3,3本身排在第三位，就不用换位置了，此时排序为1、2、3、4、5、6，前三位已固定。
  4. 从后三位中选出最小的，也就是4,4本身排在第四位，就不用换位置了，此时排序为1、2、3、4、5、6，前四位已固定。
  5. 从后二位中选出最小的，也就是5,5本身排在第五位，就不用换位置了，此时排序为1、2、3、4、5、6，前五位已固定。
  6. 此时得到的排序就是想要的排序了。

插入排序（起扑克牌法）例如要把5、6、4、3、2、1这6个数字从左往右以小到大。

  1. 把第一个数字，也就是5（第一张扑克牌），拿到手里。此时排序为5.
  2. 拿到第二个，也就是6，对比手中的5，发现比5大，把6放5右边。此时排序5、6.
  3. 拿到第三个，也就是4，对比手中的，发现比6小，然后有比5小，把4放5左边。此时排序4、5、6.
  4. 拿到第四个，也就是3，对比手中的，发现比4、5、6都小，把3放最左边。此时排序3、4、5、6.
  5. 拿到第五个，也就是2，对比手中的，发现比3、4、5、6都小，把2放最左边。此时排序2、3、4、5、6.
  6. 拿到最后一个，也就是1，对比手中的，发现比2、3、4、5、6都小，把1放最左边，此时排序1、2、3、4、5、6.
  7. 此时得到想要的排序1、2、3、4、5、6.

桶排序（强迫症受扑克牌法） 例如地上有4个A，4个2,4个K，4个3，要把它们收集起来。
随便捡一个，例如捡到A，把A放一边放好，再捡到2，又放一边放好，再捡到A，放到A放的地方叠起来，就一直这样，相同的放一起，不同的分开放。优点：速度最快，因为不用对比。缺点：需要较大的存放空间

快速排序 例如要把13、2、60、0、1、32、40、9、6这9个数字从左往右以小到大排。

  1. 以13为中心，比它小的放左边，大的放右边。此时的排序为2、0、1、9、6、13、60、32、40。此时13已结确定位置。
  2. 以2为中心，和13前面的另外几个数字对比，比它小的左边，大的右边。以60为中心，和13后面的另外几个数字对比，比它小的放左边，大的放右边。因为对比的数少很多，所以节省很多时间,此时的排序为，0、1、2、9、6、13、32、40、60。此时2,13,60位置已固定。
  3. 以0为中心，和2前面的数字对比，比它小的放左边，大的放右边。以9为中心，和2后面13前面的数字作对比，比它小的放左边，比它大的放右边。以32为中心，和13后面60前面的数字对比，比它小的放左边，大的放右边。此时的排序为0、1、2、6、9、13、32、40、60。此时所有数字都以固定
  4. 此时得到想要的排序0、1、2、6、9、13、32、40、60。

快速排序之所以快，是因为每次对比过后，都缺少一半对比的东西，减少很多对比时间。
快排有时也不快，例如每次选的中心要么是最大的要么是最小的，所有数字都跑到一边去了的时候例：654321，选了6，然后选了5，以此类推。解决此类的方法是随机选个数字，所以尽量使用随机快排。

##高度是由什么决定的？

内联元素：主要是由建议行高决定，我们只能决定字体的高度，无法决定包裹字体元素的高度，这些主要由设计这些字体的设计师决定。每种不同的字体，会使包裹它的元素的高度不同
字体三部分：基线，上部，下部，还有个建议行高。每种字体的建议行高是不一样的，文字都会基线对齐。
把一个文字设置为100px，例如“hug”，从h的最上边到g的最下边为100px。
例如把span里的文字设为100px，那么span的高度会比100px高，因为会有个建议行高，每种字体的默认行高不一样。
line-height能使文字自动垂直居中。

块级元素:高度有其内部文档流元素的高度总和决定。决定，但不相等。（假设body里什么都没有，那么他的高度是0（只有个回车不算））

##文档流是什么？
文档内元素的流动方向。
内联元素从左往右流，如果流动遇到阻碍，那就换行继续从左往右流动。
如果一个元素例如，中间类截断，假设这个元素有加边框，那么那个元素会被分开，但还是一个元素，边框也还是一个，只是头在第一行，身体在第二行。
如果被截断的是英文单词，那么英文单词并不会换行，会缩到里面去。因为中文每个字是一个东西，而英文单词也是一个东西，所以默认不会被分开。如果要想使其可以被截断：

添加css属性:word-break:break-all。

块级元素从上往下流，每一块都占一行，新的一块换新一行。
把块级元素也变成从左往右流：display：inline-block（不推荐使用，建议用floot浮动）

position fixed

position属性用于指定一个元素在文档中的定位方式。
相对定位：relative。
绝对定位：absolute（基于祖先元素定位）和fixed(基于屏幕视口定位，元素位置在屏幕滚动时不会改变，会创建新的层叠上下文，脱离文档流)。
以下是基于屏幕左上角视口定位，创建新的层叠上下文

div {
position fixed;
top 0;
left 0;
}

脱离文档流，使用position fixed后，该元素会脱离文档流，脱离文档流后，其父级元素的高度就不受其影响了，因为块元素的高度为其内部文档流高度总和，脱离后就没关系了
当使用position fixd后，此元素就不会往外扩展了，它有多大，就占用多大范围。

HTTP状态码详解

Posted on 2018-04-25 |

1xx消息

这一类型的状态码，代表请求已被接受，需要继续处理。这类响应式临时响应，只包含状态航和某些可选的响应头信息，并以空行结束。由于HTTP/1.0协议中没有定义任何1xx状态码，所以除非在某些试验条件下，服务器禁止向此类客户端发送1xx响应。这些状态码代表的响应都是信息性的，标示客户应该采取的其他行动。

2xx成功

这一类型的状态码，代表请求已成功被服务器接收、理解、并接受。

3xx重定向

这类状态码代表需要客户端采取进一步的操作才能完成请求。通常，这类状态码用来重定向，后续的请求地址（重定向目标）在本次响应的Location域中指明。

4xx客户端错误

这类的状态码代表了客户端看起来可能发生了错误，妨碍了服务器的处理。除非响应的是一个HEAD请求，否则服务器就应该返回一个解释当前错误状况的实体，以及这是临时的还是永久性的状况。这些状态码适用于任何请求方法。浏览器应当向用户显示任何包含在此类错误响应中的实体内容。
如果错误发生时客户端正在传送数据，那么使用TCP的服务器实现应当仔细确保在关闭客户端与服务器之间的连接之前，客户端已经收到了包含错误信息的数据包。如果客户端在收到错误信息后继续向服务器发送数据，服务器的TCP栈将向客户端发送一个重置数据包，以清除该客户端所有还未识别的输入缓冲，以免这些数据被服务器上的应用程序读取并干扰后者。

5xx服务器错误

表示服务器无法完成明显有效的请求。这类状态码代表了服务器在处理请求的过程中有错误或者异常状态发生，也有可能是服务器意识到以当前的软硬件资源无法完成对请求的处理。除非这是一个HEAD请求，否则服务器应当包含一个解释当前错误状态以及这个状况是临时的还是永久的解释信息实体。浏览器应当向用户展示任何在当前响应中被包含的实体。这些状态码适用于任何响应方法。

HTTP入门

Posted on 2018-04-25 |

历史

1990年，Tim Berners-Lee（李爵士）发明了用网址就能访问网页的办法，他发明了第一个网页、第一个浏览器和第一个服务器。
WWW（World Wide Web）万维网，也是李爵士发明的。
主要包含三个概念：

  1. URI，俗称网址。
  2. HTTP，文本传输协议（主要用来传输内容）。
  3. HTML，超文本标记语言。

URI的作用是能让你访问一个页面，HTTP的作用是让你能下载这个页面，HTML的作用是让你能看懂这个页面。

URI分为URL（统一资源定位符）和URN（统一资源名称），我们一般使用URL作为网址，URN的作用是通过提供一种途径，用于在特定的名字空间资源的标识，以补充网址。

URL的常见组成：

https://www.baidu.com/s?wd=hello&rsv_spt=1#5
协议/域名/路径/查询参数/锚点

DNS（域名系统）

dns作为将域名与ip地址相互映射的一个分布式数据库，能够使人更方便地访问互联网。使用53端口。

HTTP的请求与响应。

  1. 客户端负责发起请求
  2. 服务端在80端口接收请求
  3. 服务端负责返回内容（响应）
  4. 浏览器负责下载响应内容
    HTTP的作用就是知道浏览器和服务器如何沟通。

##请求的格式
看起来那么多，，格式都差不多的。

1 动词 路径 协议/版本
2 Key1: value1
2 Key2: value2
2 Key3: value3
2 Content-Type: application/x-www-form-urlencoded
2 Host: www.baidu.com
2 User-Agent: curl/7.54.0
3
4 要上传的数据

  1. 请求最多包含四部分，最少包含三部分。（也就是说第四部分可以为空）
  2. 第三部分永远是一个回车（\n）
  3. 动词有GET POST PUT PATCH DELETE HEAD OPTIONS等。
  4. 这里的路径包括[查询参数]，但不包括[锚点]。
  5. 如果你没有写路径，那么路径默认为/。
  6. 第二部分中的Content-Type标注了第四部分的格式。
  7. 如果有请求的第四部分，那么在FormData或Payload里面可以看到。

响应

请求之后，除非断网或者服务器宕机了，否则都应该能的要一个响应。
响应的格式：

1 协议/版本号 状态码 状态解释
2 Key1: value1
2 Key2: value2
2 Content-Length: 17931
2 Content-Type: text/html
3
4 要下载的内容

用Chrome查看响应

  1. 打开 Network
  2. 输入网址
  3. 选中第一个响应
  4. 查看 Response Headers，点击「view source」，点击「view source」，点击「view source」
  5. 你会看到响应的前两部分
  6. 查看 Response 或者 Preview，你会看到响应的第 4 部分

HTTP状态码，状态码是服务器对浏览器说的话。

  • 1xx消息——请求已被服务器接收，继续处理。
  • 2xx成功——请求已成功被服务器接收、理解、并接收。
  • 3xx重定向——需要后续操作才能完成这一请求。
  • 4xx请求错误——请求含有词法错误或者无法被执行。
  • 5xx服务器错误——服务器在出路某个正确请求时发生错误

什么是脚本？

    确保弹出层里的地址是SSH地址，点击Use SSH按钮，然后复制这个地址。
    打开Git Bash，进入一个目录后，运行git clone 你刚才得到的以git@github.com开头的地址,运行完会发现当前目录下多了个目录。
    进入这个目录，运行ls -la，就会看到远程目录下的所有文件了，另外还会看到.git本地仓库。
    这时可以添加文件，git add,然后git aommit了

如何上传更新

如果在本地目录有任何变动，只需按照以下顺序就能上传：

  1. git add 文件路径
  2. git commit -m “信息”
  3. git pull (增量下载，如果远程仓库有变化要使用这个把仓库的下载下来以后才能上传上去)
  4. git push（上传）
