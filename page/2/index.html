DOM

DOM是一套对文档的内容进行抽象和概念化的方法。
DOM中的”D”: Document(文档)，指XML或HTML文档。
DOM中的”O”: Object(对象)，分为三种。

  • 用户定义对象(user-defined object): 由程序员自行创建的对象。
  • 内建对象(native object): 内建在JavaScript语言里的对象。如：Array、Math、Date等。
  • 宿主对象(host object): 由浏览器提供的对象。如：window
    DOM中的”M”: Model(模型)。
    HTML文档表示为一颗树，文档的每个节点(node)在内存中都表示为对象(例：{nodeName:’HTML’})，模型把它们关联起来。
    把文档变成对象的模型就是DOM

节点属性

HTML节点由Document对象构造出，元素节点由Element对象构造出，文本节点由Text对象构造出，还有其他例如注释节点(Comment对象)等，这些节点对象共同的祖先为node。Node.prototype相当于对象中的Object.prototype (node.prototype最终也是指向Object.prototype的) ,还有其他Document.prototype（只有HTML节点是这个）,Element.prototype,Text.prototype等。
console.dir()可以输出标签对象，例console.dir(document.body),console.dir(document)就是查看html对象的属性。
页面中的节点通过构造函数(Document,Element,Text等)，构造出相对应的对象，这就是DOM的主要功能。把节点变成对象，然后操作相关对象的API。

执行环境与作用域

执行环境定义了变量或函数有权访问的其他数据，决定了它们各自的行为。每个执行环境都有一个与之关联的变量对象，环境重定义的所有变量和函数都保存在这个对象中。
全局执行环境是最外围的一个执行环境，在web浏览器中，全局执行环境被认为是window对象，因为所有全局变量和函数都是作为window对象的属性和方法创建的。
每个函数都有自己的执行环境
当代码在一个环境中执行时，会创建变量对象的一个作用域链，作用域链的用途，是保证对执行环境有权访问的所有变量和函数的有序访问。
如果这个环境是函数，则将其活动对象作为变量对象，活动对象在最开始只包含一个变量，即arguments对象（这个对象在全局环境中是不存在的）。
作用域链中的下一个变量对象来自包含（外部）环境，而在下一个变量对象则来自下一个包含对象，一直延续到全局执行环境，全局执行环境的变量对象始终都是作用域链中的最后一个对象。

  1. 首先是一个全局环境(window/global)下，有个变量a和f1
  2. 然后是它的子作用域，即函数f1的作用域，这个子作用域里有变量a和f2，
  3. 然后是f1的子作用域，函数f2的作用域下，有变量a
  4. 它们是树形结构

这里a=2,并不是声明全局变量，而是给a赋值，在当前f1函数的作用域里找不到a，它就会往父级找，然后就找到了全局环境里的a了，然后就给它赋值了
假设，从f1函数的作用域开始往上找，找到最后都没有找到a，那么它就会声明一个全局a，并给a赋值。

到底什么是数组？

定义：数组是按次序排列的一组值。
实际：用Array()构造出来的对象就是数组，是一种特殊的对象。

间歇调用和超时调用

JavaScripts是单线程语言，但它允许通过设置超时值和间歇时间值来调度代码在特定的时刻执行，前者是在指定的时间过后执行代码，而后者则是每隔指定的时间就执行一次代码。

  • 超时调用需要使用window对象的setTimeout()方法，它接收两个参数：要执行的代码和以毫秒表示的时间（即在执行代码前需要等待多少毫秒）。setTimeout(function(){alert('Hello World')},1000),1000毫秒（1秒）后执行alert('Hello World')。因为JavaScript是一个单线程的解释器，因此一定时间内只能执行一段代码。为了控制要执行的代码，就有一个任务队列。这些任务会按照将他们添加到队列的顺序执行。setTimeout()的第二个参数告诉JavaScript再过多长时间把当前任务添加到队列中。如果队列是空的，那么添加的代码会执行；如果队列不是空的，那么它就要等前面的代码执行完了以后再执行。调用setTimeout()后，该方法会返回个数值ID，表示超时调用，把这个id赋值给一个变量，然后把这个变量当做取消方法的参数就可以了。取消超时调用可用cleraTimeout()方法。
  • 间歇调用会按照指定的时间间隔重复执行代码，直至间歇调用被取消或者页面被卸载。，设置间歇调用的方法时setInterval(),可接受但参数与setTimeout()相同，取消用clearInterval()。

javascript基本类型和引用类型在内存中的值

内存基本概念

内存的生存周期：

  1. 分配所需的内存
  2. 内存的读与写
  3. 不需要时将其释放
    所有语言的内存生命周期都基本一致，不同的是最后一步在低级语言中很清晰，但是在像JavaScript等高级语言中，这一步是隐藏的、透明的。

js内存生命周期：

  1. 定义变量时就完成了内存分配
  2. 使用值的过程实际上是对分配内存进行读取与写入的操作，读取与写入可能是写入一个变量或者一个对象的属性值，甚至传递函数的参数。
  3. 内存的释放依赖GC机制（高级语言解释器嵌入的“垃圾回收器”）。

数据和代码在内存中的存放

js程序在运行的时候，需要内存空间存放代码和数据，所以分为代码区和数据区。而数据区又分为栈(stack)和堆(heap)两种内存空间。

  • 栈(stack)是有结构的，每个块区按照一定次序存放（先进后出），sack中主要存放一些基本的变量和对象的引用，存在栈中的数据大小与生存期必须是确定的。可以明确知道每个区块的大小，因此，stack的寻址速度要快于heap。它的存储方式是一行一行的和栈类似。简单数据类型都存在这（6种）。
  • 堆(heap)是没有结构的，数据可以任意存放。heap用于复杂数据类型（引用类型）分配空间，例如数组对象、object对象。它的存储方式跟堆类似。
    例var a = 1,首先变量a提升，此时a等于undefined，然后a放在代码区，1放在数据区的栈里。在js内部，所有数字都是以64位浮点数形式储存，所以这个1在内存里占了64位。ECMAScript规定。
    复制基本类型：var b = 2,也和上面一样，当b=a，那么就把a对应的在栈里的64位浮点数形式的1覆盖在b对应的栈的位置。
    js在存字符的时候，每个字符占16位。ECMAScript规定
    存对象：var a = {name:'luo',age:18},那么会把a放在代码区，然后a对应的栈内存里有个ADDR（地址），那个地址是堆内存中的一块空间的地址，这块空间大小可以变化，就可以不受影响的增加或减少对象里的东西了。
    复制对象：相当于把变量对应的栈里面的ADDR复制给另一个变量。

引用：当复制保存着对象的某个变量时，操作的是对象的引用（就是那个地址）。但在为对象添加属性时，操作的是实际的对象。
关于循环引用var a,a={self:a},a.self等于undefined。也就是得到的对象是{self:undefined},因为第二步把对象赋值给a的时候，对象里的a等于undefined，运算是从右到左的。
循环引用应该是var a = {},a.self=a,会得到一个{self:{…}},点开发现里面也是一个{self:{…}}，一直循环。因为我们把a的值赋值给self的value，然后我们打开对象a，发现里面的栈有个地址，说明是个对象，按着这个地址找到堆里的空间，发现里面的self对应的又是个地址，再按着这个地址，一直循环，一直都在那个空间里找。

引用类型面试题：
输入：var a={n:1};var b=a;a.x=a={n:2}。
输出alert(a.x);-->undefined,alert(b.x);-->[object Object]。。
首先前两步是没有问题的，生成一个对象，赋值给a，然后把a的地址赋值给b，此时a和b在stack中的值（地址）是一样的，假设为ADDR100,对应的heap中的对象也是一样的。
然后a.x=a={n:2},假设{n:2}的地址为ADDR200，浏览器读数据是从左到右的，运算是从右到左的。所以此时a的值为ADDR100，右边运算赋值后给中间a后，中间的a后此时对应的stack中的值为ADDR200,左边的a还是ADDR100，然后把中间那个a，也就是ADDR200,赋值给ADDR100中的对象中的key为x的value。
那么此时a是ADDR200那里的的对象，里面的对象为{n:2},b为ADDR100那里的对象，里面的对象为{n:1,x:{n:2}},第三步的赋值只是赋给了ADDR100对应的对象里面的key为x的value。
当使用alert(a.x)，会调用toString方法。所以alert(a.x);-->undefined,alert(b.x);-->[object Object]。

垃圾回收机制（GC）

Javascript具有自动垃圾回收机制。
如果一个对象没有被引用，他就是垃圾，将被回收。
var a={name:'a'};var b={name:'b'};a=b，a就变成垃圾，将被回收，至于什么时候回收是不确定的。
var fn = function(){};document.body.onclick=fn;fn=null;请问这个函数是不是垃圾？
答：不是，当函数赋值给fn的时候，fn的stack指向那个对象，fn赋值给document.body.onclick时，document指向body,body指向onclick，onclick指向那个对象。当fn等于null，document那条连接并没有改变，对象还是被引用。
如果onclick等于null的话，那么它就是垃圾了。
如果把页面关闭的话，它也变成垃圾了。因为页面关了，相当于document不见了，然后它们内部只有body–>onclick–>function(){}了。
在IE6里面这三个不算垃圾，它会把这三个对象一直留着，只要不关闭整个浏览器就会一直留着，这样垃圾会越来越多。我们要把这个垃圾丢掉的话，只能运行window.onunload = function(){document.body.onclick = null}这种代码，把所有的事件监听赋值为空，所有的，这就是内存泄漏。
内存泄漏就是由一些浏览器引起的bug，使一些该被标记为垃圾的东西没有被标记为垃圾，然后内存就会被永久的占用着，浏览器会占用越来越多内存，会变慢，甚至崩溃。

深拷贝和浅拷贝概念

深拷贝：var a=1;var b=a;b变，不影响a。所以一般说深拷贝和浅拷贝的时候不考虑基本类型，因为赋值都是深拷贝。我们只说对象。
浅拷贝：var a={name:'a'};var b=a;b.name='b'；此时a={name:’b’},把a复制给b后，b改变，a也改变，这就是浅拷贝。
深拷贝：a和b在stack里有不同的地址，在heap里的空间也不一样，假设在heap里的对象中还有地址，也必须是不同的，这是两条分开，独立，互不影响的两条线。b变，a不变。这就是深拷贝。

响应式Web设计-viewport

什么是viewport?

viewport是用户网页的可视区域，翻译为中文叫做“视区”。
<meta name="viewport" content="width=device-width, initial-scale=1.0,user-scalable=no,maximum-scale=1.0,minimum-scale=1.0">
宽度=设备宽度，不允许缩放，所有比例都是1比1.

六个属性：

  1. width:控制viewport的大小，可以指定的一个值，如600，或者特殊的值，如device-width为设备的宽度（相当于100%的css像素）
  2. height：和width相对应，指定高度。
  3. initial-scale：初始缩放比例，也即是当页面第一次load的时候缩放比例。1.0，就是1：1
  4. maximum-scale：允许用户缩放到的最大比例。1.0
  5. minimum-scale：允许用户缩放到的最小比例。1.0
  6. user-scalable：用户是否可以手动缩放。yes或no

touch触摸事件

触摸事件的种类，有以下几种。可以通过TouchEvent。type属性，查看到底发生的是哪一种事件。

  1. touchstart:用户开始触摸时触发，它的target属性返回发生触摸的元素节点。
  2. touchend:用户不在接触触摸屏时（或移除屏幕边缘时）触发，它的target属性与touchstart事件一致的，就是开始触摸时所在的元素节点。它的changedTouches属性返回一个TouchList实例，包含所有不在触摸的触摸点（即Touch实例对象）
  3. touchmove:用户移动触摸点时触发，它的target属性与touchstart事件一致。如果触摸的半径、角度、力度发生变化，也会触发该事件。
  4. touchcancel:触摸点取消时触发，比如在触摸区域跳出一个情态窗口（modal window）、触摸点离开了文档区域（进入浏览器菜单栏）、用户的触摸点太多，超过了支持的上限（自动取消早先的触摸点）。

查看设备是否支持触摸(特性检测)

  1. canvas.ontouchstart，如果显示为null，则为可触摸设备，表示支持该功能，不支持显示为undefined。canvas为变量名。
  2. 查看ontouchstart是否body的一个属性。例：'ontouchstart' in document.body显示true，代表是，那么可触摸。
  3. document.body.touchstart == undefinede为非触屏设备。document.body.touchstart == null为触屏设备。
    in，用于查看什么属性或者什么值在不在xxx里面，例如：查看一个key在不在哈希里。'key' in hash，在就输出true，不在就输出false。

mouse事件

  • onmousedown:在用户按下了任意鼠标按键时触发。不能通过键盘触发这个事件。
    document.onmousedown = function(x){ console.log(x) }

  • onmousemove:当鼠标指针在元素内部移动时重复触发。不能通过键盘触发这个事件。
    document.onmousemove = function(x){ console.log(x) }

  • onmouseup:当用户释放鼠标按键时触发。不能通过键盘触发这个事件。
    document.onmousemove = function(x){ console.log(x) }

这些事件，前面可加on可不加on，即onmousedown和mousedown一样

画线

  1. 定义一个变量赋值为false.
  2. 按下鼠标任意键，即触发mousedown时，给变量赋值true
  3. 在鼠标指针滑动，即触发mousemove时，判断变量是否为true，即是否已按下鼠标按键，如果是，执行代码，如果不是，就不执行。
  4. 当鼠标释放时，即触发mouseup事件时，非变量赋值false，即停止鼠标指针滑动时的执行的代码。
