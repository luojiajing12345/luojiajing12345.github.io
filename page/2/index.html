无缝轮播

发表于 2018-06-13

思路

假设有三张图，开始把第一张图在前面，后面跟着第二张图，然后到第二张图时，第三张图在后面，到第三张图时，又把第一张图放第三张图后面，如此循环。

代码思路

一个大div，一个小div，小div里有4张图，这4张图绝对定位于同一个位置，设置3个类样式，一个为显示在中间，一个为在右边，一个为在左边，设置translateX，在左边就-100%，右边就100%，设置完这三个样式，然后设置js切换着三个样式，当第一张图片滑动过去后切换状态，跑到右边，第二张跑到中间，然后第二张滑过去，先到左边然后又到右边，第三张滑到中间，使用一个增量然后使它一直从等于从1-4，循环，然后这4张图片就一直这三个样式切换，实现了html，css，js分离。

jQuery基础——样式篇

初始jQuery

  1. jQuery 分 2 个系列版本 1.x 与 2.x，主要的区别在于 2.x 不再兼容 IE6、7、8浏览器。。jQuery 每一个系列版本分为：压缩版(compressed) 与 开发版(development)，我们在开发过程中使用开发版（开发版本便于代码修改及调试），项目上线发布使用压缩版（因为压缩版本体积更小，效率更快）。jQuery是一个JavaScript脚本库，不需要特别的安装，只需要我们在页面 /<head/> 标签内中，通过 script 标签引入 jQuery 库即可。
  2. $(document).ready()的作用是等页面的文档中的节点都加载完毕后，再执行后续的代码，因为我们在执行代码的时候，可能会依赖页面的某一个元素，我们要确保这个元素真正的被加载完毕后才能正确的使用。
  3. 通过原生DOM模型提供的方法获取的DOM元素就是一个DOM对象，var $p = $('#imooc')通过$(‘#imooc’)方法会得到一个$p的jQuery对象，$p是一个伪数组对象，这个对象里面包含了DOM对象的信息，然后封装了很多操作方法。
  4. 标准的JavaScript操作DOM与jQuery操作DOM的对比：一、通过jQuery方法包装后的对象，是一个类数组对象。它与DOM对象完全不同，唯一相似的是它们都能操作DOM。二、通过jQuery处理DOM的操作，可以让开发者更专注业务逻辑的开发，而不需要我们具体知道哪个DOM节点有那些方法，也不需要关心不同浏览器的兼容性问题，我们通过jQuery提供的API进行开发，代码也会更加精短。
  5. var $div = $('div');var div=$div[0],jQuery对象转化成DOM对象。用jQuery找到所有的div元素，因为jQuery对象也是一个数组结构，可以通过数组下标索引找到第一个div元素。var $div = $('div');var div=$div.get(0)，使用jQuery对象自身提供的一个.get()方法直接访问jQuery对象中相关的节点。
  6. 如果传递给$(DOM)函数的参数是一个DOM对象，jQuery方法会把这个DOM对象给包装成一个新的jQuery对象。var div = document.getElementsByTagName('div');var $div = $(div),通过getElementsByTagName获取到所有div节点的元素，结果是一个dom合集对象，再通过$(div)方法转化成jQuery对象。

HTML、CSS、JS，内容、样式与行为分离。

如果用html写样式，会多了很多标签，有的标签是表现样式的，有的标签是表现结构的，很难看出逻辑结构。

轮播的思路

  1. 滚来滚去
  2. 用局部画面骗用户（就是把图片排成一排，不需要的图片挡住，要的显示出来就可以了。）

DOM

DOM是一套对文档的内容进行抽象和概念化的方法。
DOM中的”D”: Document(文档)，指XML或HTML文档。
DOM中的”O”: Object(对象)，分为三种。

  • 用户定义对象(user-defined object): 由程序员自行创建的对象。
  • 内建对象(native object): 内建在JavaScript语言里的对象。如：Array、Math、Date等。
  • 宿主对象(host object): 由浏览器提供的对象。如：window
    DOM中的”M”: Model(模型)。
    HTML文档表示为一颗树，文档的每个节点(node)在内存中都表示为对象(例：{nodeName:’HTML’})，模型把它们关联起来。
    把文档变成对象的模型就是DOM

console.dir()可以输出标签对象，例console.dir(document.body),console.dir(document)就是查看html对象的属性。
页面中的节点通过构造函数(Document,Element,Text等)，构造出相对应的对象，这就是DOM的主要功能。把节点变成对象，然后操作相关对象的API。

执行环境与作用域

执行环境定义了变量或函数有权访问的其他数据，决定了它们各自的行为。每个执行环境都有一个与之关联的变量对象，环境重定义的所有变量和函数都保存在这个对象中。
全局执行环境是最外围的一个执行环境，在web浏览器中，全局执行环境被认为是window对象，因为所有全局变量和函数都是作为window对象的属性和方法创建的。
每个函数都有自己的执行环境
当代码在一个环境中执行时，会创建变量对象的一个作用域链，作用域链的用途，是保证对执行环境有权访问的所有变量和函数的有序访问。
如果这个环境是函数，则将其活动对象作为变量对象，活动对象在最开始只包含一个变量，即arguments对象（这个对象在全局环境中是不存在的）。
作用域链中的下一个变量对象来自包含（外部）环境，而在下一个变量对象则来自下一个包含对象，一直延续到全局执行环境，全局执行环境的变量对象始终都是作用域链中的最后一个对象。

  1. 首先是一个全局环境(window/global)下，有个变量a和f1
  2. 然后是它的子作用域，即函数f1的作用域，这个子作用域里有变量a和f2，
  3. 然后是f1的子作用域，函数f2的作用域下，有变量a
  4. 它们是树形结构

这里a=2,并不是声明全局变量，而是给a赋值，在当前f1函数的作用域里找不到a，它就会往父级找，然后就找到了全局环境里的a了，然后就给它赋值了
假设，从f1函数的作用域开始往上找，找到最后都没有找到a，那么它就会声明一个全局a，并给a赋值。

实际：用Array()构造出来的对象就是数组，是一种特殊的对象。
数组有自己的公用属性(Array.prototype),函数也有自己的公有属性(Function.prototype)。
数组和函数和对象有什么不一样？或者说为什么对象里还有分数组和函数？
因为它们的公有属性不一样，数组的proto指向Array.prototype,它拥有自己的API，函数的proto指向Function.prototype,也拥有函数特有的API，而对象的proto指向Object.prototype，只有对象的API，没有其他的API

间歇调用和超时调用

JavaScripts是单线程语言，但它允许通过设置超时值和间歇时间值来调度代码在特定的时刻执行，前者是在指定的时间过后执行代码，而后者则是每隔指定的时间就执行一次代码。

  • 超时调用需要使用window对象的setTimeout()方法，它接收两个参数：要执行的代码和以毫秒表示的时间（即在执行代码前需要等待多少毫秒）。setTimeout(function(){alert('Hello World')},1000),1000毫秒（1秒）后执行alert('Hello World')。因为JavaScript是一个单线程的解释器，因此一定时间内只能执行一段代码。为了控制要执行的代码，就有一个任务队列。这些任务会按照将他们添加到队列的顺序执行。setTimeout()的第二个参数告诉JavaScript再过多长时间把当前任务添加到队列中。如果队列是空的，那么添加的代码会执行；如果队列不是空的，那么它就要等前面的代码执行完了以后再执行。调用setTimeout()后，该方法会返回个数值ID，表示超时调用，把这个id赋值给一个变量，然后把这个变量当做取消方法的参数就可以了。取消超时调用可用cleraTimeout()方法。
  • 间歇调用会按照指定的时间间隔重复执行代码，直至间歇调用被取消或者页面被卸载。，设置间歇调用的方法时setInterval(),可接受但参数与setTimeout()相同，取消用clearInterval()。

全局对象window

当我们没有写任何代码的时候，里面就有个window对象，对象里有一堆函数(对象)。
浏览器默认全局对象window，无论在哪个网页都可以访问一个对象那就是window。
在浏览器中，window对象有双重角色，即是通过Javascript访问浏览器窗口的一个借口，有时ECMAScript规定的Globla对象。

由于window对象同时扮演着ECMAScript中的Global对象的角色，因此所有在全局作用域中声明的变量、函数都会变成window对象的属性和方法。
window对象下面一堆函数，对应一堆对象，对象下面又有对象，所有window下面很多对象。window里的函数前面可以不加window，例如window.alert(),直接alert()。
window下面的属性分为两类。

  1. ECMAScript规定的函数。
    parseInt、parseFloat、Number()（默认0）、String()（默认’’）、boolean()（默认false）、Objict(){默认{}}，window.setTineout(function(){} , time)(在指定时间后执行这个函数)
  2. 私有的函数（Chrome/ie/Fireox，像这些浏览器独有的，或者只有哪两个有的）。
    alert(弹框提示)、prompt(用户填写)、comfirm(确认)、console(开发者)。像这些浏览器私有的，每个浏览器的实现效果都可能不一样，因为没有标准，例如alert(),每个浏览器弹出的位置，大小，形状等都不一样。
    document(文档)有自己的规范（W3C规定），规定了能对文档执行的一切操作。DOM.
    history(浏览器),获取浏览器的历史，访问记录什么的。BOM.

简单类型和对象的区别

Number()

Number(‘1’)
var n = new Number(1);1相当于{valueof():1}
var n1 = 1;var n2=new Number(1)这两个的区别在于内存不同，n1就是在stack里有个数值1，n2是在stack里有个ADDR，然后指向heap里的对象{[[PrimitiveValue]]：1}，原始值为1，里面是个哈希。我们调用n2.valueOf(),能获取原始的这个1，用n2.toString()能获取它的字符串。没有人用第二种方法。
但第一种实际上是不能用toString()的，但js之父设计了一个临时转换的技巧，当n1.toString()的时候，会出现一个临时对象，调用temp=new Number(n1);temp.toString();然后这个temp就像不存在一样，就获取到它转换成字符串的了
所以就算我们使用第一种那种简单的赋值，也能使用第二种里的所有属性。
var n=1;n.xxx=2;n.xxx等于几？undefined。首先在stack里有个1，然后第二句的时候，stack里有个temp的ADDR，然后会在heap里建个对应的临时对象，n的值是1，它的属性xxx对应2，{xxx:2},运行这句以后temp会被回收，临时对象也会一起被回收，第三句的时候，没有这个xxx，会新建一个，新建的对象的没有xxx属性所以等于undefined。简单点说就是运行第二句会有个临时新对象，并把2赋值给它的xxx属性，运行完后销毁，第三局又新建个对象。

共有属性（原型）

  • 上面几个对象的共有属性，像那些对象，有一些属性是每个都有的，例如：toString(),valueOf()等，主要作用是不要浪费内存。像这些共有属性都在隐藏的proto属性里面。当对象调用toString()方法时，会先在自己的属性里找有没有这个属性，如果没有找到，就会去proto里面找，找到后就会调用。
  • 以上所说的是对于常规对象，对于其他对象，例如Number对象，String对象，Boolean对象。它们会有一些自己才有的公有属性，例如Number对象的特有的toFixed(),toExponential()，String对象的length()等，它们首先是在自己的属性里找，然后在第一个proto里（类型对象的共有属性），然后再到所有对象的共有属性（第二个proto）里找，如果都找不到就会输出undefined。
  • 如果这些共有属性没有被人引用的话，是会被垃圾回收掉的，所以有东西在一直引用它。Object.prototype就是共有属性（原型），Number.prototype就是Number对象的共有属性。obj.__proto__===Object,prototype,Num.__proto__===Number.prototype,Num.__proto__.__proto__====Object.prototype。
  • Number对象，String对象，Boolean对象都有两个proto,object只有一个（为所有对象的共有属性）。这种几个结点连接一个结点也叫原型链。
  • 当我们没有写任何代码的时候，里面就有个window对象（浏览器在内存里初始化好了），对象里有一堆函数(对象)。例如：Number函数（对象），String函数（对象），Boolean函数（对象），这些对象里有个prototype属性，这个属性引用这它们自己类型的共用属性（例Number函数的prototype引用着Number对象的共用属性）,然后这些属性里有个proto引用着总对象的共用属性。
  • 当我们写了一句代码var a=new String('hi')，a对应的stack里存了一个heap里的地址，然后在heap里面有块空间，写着
    {0:’h’,1:’i’,length:2,proto:String},这个proto引用String的共用函数。window里的String里的prototype和s里的proto都指向String的共用属性，它们没有什么很大的区别。唯一的区别就是，前者是浏览器写好的，一开始就有的，String.prototype是String的共用属性的引用（如果没有这个，String会被垃圾回收掉）。后者是自己写了代码后才有的，a.proto是String的共用属性的引用（因为我们要用到它的共用属性）
