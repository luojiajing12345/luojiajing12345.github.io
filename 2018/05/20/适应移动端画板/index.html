响应式Web设计-viewport

什么是viewport?

viewport是用户网页的可视区域，翻译为中文叫做“视区”。
<meta name="viewport" content="width=device-width, initial-scale=1.0,user-scalable=no,maximum-scale=1.0,minimum-scale=1.0">
宽度=设备宽度，不允许缩放，所有比例都是1比1.

六个属性：

  1. width:控制viewport的大小，可以指定的一个值，如600，或者特殊的值，如device-width为设备的宽度（相当于100%的css像素）
  2. height：和width相对应，指定高度。
  3. initial-scale：初始缩放比例，也即是当页面第一次load的时候缩放比例。1.0，就是1：1
  4. maximum-scale：允许用户缩放到的最大比例。1.0
  5. minimum-scale：允许用户缩放到的最小比例。1.0
  6. user-scalable：用户是否可以手动缩放。yes或no

touch触摸事件

触摸事件的种类，有以下几种。可以通过TouchEvent。type属性，查看到底发生的是哪一种事件。

  1. touchstart:用户开始触摸时触发，它的target属性返回发生触摸的元素节点。
  2. touchend:用户不在接触触摸屏时（或移除屏幕边缘时）触发，它的target属性与touchstart事件一致的，就是开始触摸时所在的元素节点。它的changedTouches属性返回一个TouchList实例，包含所有不在触摸的触摸点（即Touch实例对象）
  3. touchmove:用户移动触摸点时触发，它的target属性与touchstart事件一致。如果触摸的半径、角度、力度发生变化，也会触发该事件。
  4. touchcancel:触摸点取消时触发，比如在触摸区域跳出一个情态窗口（modal window）、触摸点离开了文档区域（进入浏览器菜单栏）、用户的触摸点太多，超过了支持的上限（自动取消早先的触摸点）。

查看设备是否支持触摸(特性检测)

  1. canvas.ontouchstart，如果显示为null，则为可触摸设备，表示支持该功能，不支持显示为undefined。canvas为变量名。
  2. 查看ontouchstart是否body的一个属性。例：'ontouchstart' in document.body显示true，代表是，那么可触摸。
  3. document.body.touchstart == undefinede为非触屏设备。document.body.touchstart == null为触屏设备。
    in，用于查看什么属性或者什么值在不在xxx里面，例如：查看一个key在不在哈希里。'key' in hash，在就输出true，不在就输出false。

获取触摸位置

在触摸事件属性里，在touches:TouchList里的第0项，有那个触摸点的clienX和clineY坐标，为什么是第0项呢？因为在触摸设备里，可能有二点触控，三点触控，五点触控，这是一个触摸点列表集合。
canvas.ontouchstart = function(a){ var x = a.touches[0].clientX var y = a.touches[0].clientY }
