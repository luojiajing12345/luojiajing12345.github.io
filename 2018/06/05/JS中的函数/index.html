JS中的函数

发表于 2018-06-05

函数的五种声明方式

  1. 具名函数

  2. 匿名函数

  3. 具名函数赋值给变量

  4. window.Function(基本不会用)

  5. 箭头函数

函数的name属性

所有的函数都有一个name属性，改属性保存的是该函数名称的字符串。
没有名称的函数也仍然有name属性，该属性值会返回实际的函数名。（es6）
另外对于Funtion构造函数返回的函数实例，name属性的值为”anonymous”。

  1. 具名函数，它的name属性，就是它的名字

  2. 匿名函数，它的name属性，就是变量的名字

  3. 具名函数赋值给变量,它的name属性，就是函数的名字

  4. 用Function，它的name属性，是”anonymous”(匿名的)

  5. 箭头函数,它的name属性，就是变量的名字

如何调用函数

函数是什么

函数是一段可以反复调用的代码块。函数还能接受输入的参数，不同的参数会返回不同的值。
函数是个对象，它可以执行一段代码，可以执行一段代码的对象就是函数

函数在内存中

声明一个函数var f(x,y){return x+y},stack中有个对应f的地址（假设为ADDR100）,ADDR指向heap中的一块空间，假设里面的属性值为params:['x','y'],fbody:'return x+y',__proto__,proto指向公有对象，公有对象中有个call()方法，使用这个方法就可以调用fbody。

Function.prototype三个重要的方法

call()
apply()
bind()

函数的两种调用方法

function f(x,y){return x+y}

  1. f(1,2)(糖)
  2. f.call(undefined,1,2)(真正的调用方法，硬核)

eval

eval命令的作用是，将字符串当作语句执行。
eval('var a=1')
a //1
eval('1+1') //2

this和arguments

  1. call的第一个参数可以用this得到
  2. call的后面的参数可以用arguments得到
  3. 可以理解为f.call(this,arguments)
    示例：

在正常模式下，如果this是undefined或null，那么它会浏览器会自动把this变成window，如果是数字，会变成Number对象，字符串会变成String对象，布尔会变成Boolean对象，Object会变成个Object对象。
在严格模式(加了’use strict’)下,f.call()的第一个参数是什么，this就是什么。
this的存在是为了仿java，没有什么很大的用处

arguments是一个伪数组，就是它的原型链里没有Array.prototype，但有0,1,2,3,length这些属性。

递归

递归函数是在一个函数通过名字调用资深的情况下构成的。

call stack调用栈

在进行递归的时候，会把没算出的式子放进stack中，但是stack里面是有限的，超出了的话就会报错（stack overflow(栈溢出)）。

执行环境与作用域

执行环境定义了变量或函数有权访问的其他数据，决定了它们各自的行为。每个执行环境都有一个与之关联的变量对象，环境重定义的所有变量和函数都保存在这个对象中。
全局执行环境是最外围的一个执行环境，在web浏览器中，全局执行环境被认为是window对象，因为所有全局变量和函数都是作为window对象的属性和方法创建的。
每个函数都有自己的执行环境
当代码在一个环境中执行时，会创建变量对象的一个作用域链，作用域链的用途，是保证对执行环境有权访问的所有变量和函数的有序访问。
如果这个环境是函数，则将其活动对象作为变量对象，活动对象在最开始只包含一个变量，即arguments对象（这个对象在全局环境中是不存在的）。
作用域链中的下一个变量对象来自包含（外部）环境，而在下一个变量对象则来自下一个包含对象，一直延续到全局执行环境，全局执行环境的变量对象始终都是作用域链中的最后一个对象。

  1. 首先是一个全局环境(window/global)下，有个变量a和f1
  2. 然后是它的子作用域，即函数f1的作用域，这个子作用域里有变量a和f2，
  3. 然后是f1的子作用域，函数f2的作用域下，有变量a
  4. 它们是树形结构

这里a=2,并不是声明全局变量，而是给a赋值，在当前f1函数的作用域里找不到a，它就会往父级找，然后就找到了全局环境里的a了，然后就给它赋值了
假设，从f1函数的作用域开始往上找，找到最后都没有找到a，那么它就会声明一个全局a，并给a赋值。
